# AGPL 授權風險與防範 — 互動式學習網站｜版面架構與視覺設計藍圖

> 對象：金融資訊人員（IT / 開發者）　｜　雙重用途：① 自學互動網站　② 資深智財律師「兩小時講座」投影骨架
> 設計準則：**拒絕長篇大論**，每個知識點 = 一個儀表板 / 視覺化圖表 / 互動元件。

本檔說明每個區塊「為什麼這樣設計、視覺如何呈現、互動如何運作」。實作見專案根目錄 `index.html`（單檔、零外部相依、可離線開啟，亦可直接投影）。

## 0. 整體資訊架構（IA）

## 1. Hero + 綜合曝險儀表板（Dashboard）

**視覺呈現**
- **半圓儀表 (Gauge)**：指針指向「極高」，四色弧段對應風險級距。一眼傳達「對金融業＝極高曝險」。
- **三張 KPI 卡 + 進度條**：
  - 觸發後果 `100%`（須提供完整對應原始碼）
  - 傳染途徑 `§13`（封堵 SaaS 漏洞）
  - AI 引入風險 `↑↑↑`
- **紅色 callout 一句話摘要**：對一般 SaaS 是「需謹慎」，對金融機構是「**預設禁用、例外才放行**」。

**設計理由**：開場不講定義，先用儀表板把「嚴重性」量化、視覺化，建立焦慮再給知識。

## 2. 區塊 01｜授權光譜（License Spectrum）

**視覺呈現（雙頁籤 Tab 切換）**
- **雷達圖**：以「商用自由度／可閉源／網路觸發／傳染強度／金融適用」五軸，疊三條折線（MIT·Apache / LGPL·MPL / AGPL）。AGPL 在「網路觸發」「傳染強度」滿格、「金融適用」幾乎為零——形狀對比即結論。
- **義務熱力矩陣**：列＝各授權、欄＝五種義務，格子綠（無）→紅（強制）。**最後一欄「網路使用即觸發」只有 AGPL 是紅的**，視覺上孤立出 AGPL 的危險性。

**設計理由**：用「形狀」與「顏色孤立」取代條文比較表，工程師 3 秒看懂相對風險。

## 3. 區塊 02｜§13 網路互動條款（觸發機制）

**視覺呈現**
- **三段流程節點圖**：引入 AGPL → 修改整合 → 對外網路服務（最後一格標紅 danger），下接一條紅色「義務啟動」總結節點。
- **GPL vs AGPL 對照框**：左 `GPL §0`（伺服器執行不算散布／綠）vs 右 `AGPL §13`（網路互動即視同散布／紅）。

**核心訊息（律師金句）**：「GPL 防的是『拿走』；AGPL 防的是『躲在伺服器後面拿走』。金融業最大的資產，剛好躲在伺服器後面。」

## 4. 區塊 03｜金融場景風險熱力圖（互動）

**視覺呈現**
- **二維熱力圖**：縱軸＝金融系統類型（內部後台 → 行員系統 → 網銀/App → 開放 API → 核心交易/風控）；橫軸＝整合方式（你是純客戶端 → 獨立行程 → 動態連結 → 靜態/同行程，越右越緊耦合）。
- 格子數值 1–5、四色漸層；**點擊任一格** → 下方詳情卡顯示風險等級徽章 + 處置建議。

**設計理由**：把「同元件、不同用法、風險天差地別」這個最常被誤解的觀念，做成可操作的座標定位。講座時請聽眾「喊出自己系統在哪一格」。

---

## 5. 區塊 04｜生成式 AI 污染鏈

**視覺呈現**
- **三段流程圖**：AI 補全/貼上 → 無授權標頭（SCA 難偵測）→ 進入金融核心（隱性觸發 §13）。
- **雙欄卡**：「為什麼特別危險」vs「對應防線」並排對照。

**關鍵洞見**：AI 吐出的片段常**無 LICENSE/SPDX 標頭**，傳統 metadata 掃描失效，必須靠**片段級相似度比對**才能偵測；且「AI 寫的」不能免責。

## 6. 區塊 05｜互動決策樹（自我診斷）

**視覺呈現**
- **點擊式決策樹**：4 層提問（是否 AGPL → 是否對外網路服務 → 整合方式 → 能否商業授權/替換），頂部以麵包屑顯示已選路徑，葉節點輸出**彩色判讀卡**（綠/橙/紅徽章 + 處置建議）。
- 鍵盤 `G` 一鍵重置；判讀卡含「↺ 重新評估」。

**設計理由**：給工程師在 code review / 選型當下 30 秒自評。講座時分組各跑一條路徑、上台報告，是最佳互動橋段。

## 7. 區塊 06｜防範成熟度儀表板 + 導入檢核清單

**視覺呈現**
- **六軸雷達圖**：政策／SCA（片段級）／CI-CD 門禁／SBOM／AI 工具治理／法務流程。疊兩條線：灰＝多數金融機構現況、綠＝目標成熟度，缺口一目了然。
- **互動檢核清單 + 進度條**：六項可勾選，即時計算「防線完成度 %」並推動進度條。

**縱深防線六項（帶得走的行動）**：授權政策（黑白名單）→ 片段級 SCA → CI/CD 授權門禁（fail build）→ SBOM 可追溯 → AI 工具治理（公開碼過濾＋來源宣告）→ 例外審批與法務簽核。

## 8. 區塊 07｜授權變更與爭議時間軸

**視覺呈現**：垂直 timeline，節點標記重大事件（2007 AGPLv3 發布、2012 MongoDB 採 AGPL、2018 轉 SSPL、2021 Grafana/Elastic 改授權、2022→ 生成式 AI 普及）。紅點＝高衝擊事件、金點＝里程碑。

**核心訊息**：開源授權是「**會變動的商業武器**」，今天寬鬆的元件明天可能轉 AGPL/SSPL；依賴前須評估「授權變更風險」。

## 技術實作備註

- **單檔 `index.html`**：內嵌 CSS/JS，**零外部相依、零 CDN**，可離線開啟與投影（金融內網友善）。
- 所有圖表（儀表、雷達、熱力圖、決策樹）皆以**原生 SVG / DOM** 動態繪製，無第三方函式庫，授權乾淨、易於內部審查。
- RWD：桌機多欄、平板/手機自動降為單欄，方便講座掃 QR 用手機操作。
