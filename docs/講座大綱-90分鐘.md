## ① 開場 · 四個真實踩雷案例（00:00–00:15）

> 開場鉤子：「請舉手——上週用過 AI 補全寫程式的人？……幾乎全部。那你能保證它吐出的每一行，都不是某個 AGPL 專案的複製品嗎？」

1. **Artifex v. Hancom（2017，美國加州北區法院）**
   - Ghostscript 採「AGPL 或商業授權」雙軌；Hancom 兩邊都不遵守。
   - 法院：違反 GPL/AGPL **可同時構成著作權侵權與違約**。開源授權「告得贏」。
   - 陷阱 ①：**免費版 ≠ 可商用**（雙重授權的免費軌常是 AGPL）。

2. **iText PDF（金融報表的隱藏地雷）**
   - 普及的 PDF 函式庫，金融業常用於報表、對帳單、KYC 表單；開源版是 **AGPL**。
   - 工程師「找個套件就用」→ 對外服務後整套落入揭露範圍，事後常需補繳商業授權費。
   - 陷阱 ②：**常用 ≠ 安全**。

3. **GitHub Copilot 集體訴訟（Doe v. GitHub, 2022）**
   - 主張 AI 逐字產出他人程式碼卻不附授權與出處（涉 DMCA §1202 移除著作權管理資訊）。
   - 污染從「選型問題」變成「逐行問題」，傳統 license 掃描看不到。
   - 陷阱 ③：**AI 產 ≠ 免責**（責任仍在公司）。

4. **MongoDB / Elastic / Grafana 授權變更潮**
   - 2018 MongoDB→SSPL、2021 Elastic→SSPL（催生 AWS OpenSearch）、2021 Grafana→AGPLv3。
   - 陷阱 ④：**當初寬鬆 ≠ 永遠寬鬆**（授權是會變的商業武器）。

**收束**：四案 → 四陷阱（免費≠可商用、常用≠安全、AI產≠免責、寬鬆≠永遠）。

## ② 授權光譜與 copyleft 基本盤（00:15–00:30）

- 一條光譜：`MIT/BSD → Apache-2.0 → LGPL/MPL → GPL-3.0 → AGPL/SSPL`，越右殺傷力越大。
- 三類心法：寬鬆（可閉源，首選）／弱傳染（改它才回饋，可控）／強傳染（構成衍生即揭露）。
- 類比：MIT＝「拿去用，記得謝我」；AGPL＝「拿去用，連你後來蓋的房子都要一起公開」。

---

## ③ AGPL §13 網路互動條款（00:30–00:45）

- **GPL §0**：伺服器執行、只回結果 = 不算散布 → 不需給原始碼（SaaS 漏洞）。
- **AGPL §13**：網路遠端互動 = 視同散布 → 必須對每位使用者提供完整對應原始碼。
- **金融命門**：系統多為伺服器端/雲端（網銀、App、開放 API），核心是專有交易/風控/定價邏輯，外加監理與個資義務 → 揭露原始碼形同自曝核心資產且牴觸資安。
- 金句：「GPL 防的是拿走，AGPL 防的是躲在伺服器後面拿走。」
- 等級結論：對金融機構，AGPL =「**預設禁用、例外才放行**」。

## ④ AI 時代的開源應用：Claude / Copilot / RAG / MCP（00:45–01:05）★本場重點

- **新常態 stack**：基礎模型（Claude 等以 API/雲端服務呼叫）＋ RAG/向量庫 ＋ MCP 工具伺服器 ＋ AI 編碼助手。
- **關鍵區分**：「**用 AI 服務**」（通常無 copyleft 問題）vs「**散布 AI 產出的程式碼**」（污染來源）。
- **向量庫 / MCP server**：勿假設都寬鬆，引入前逐一查 LICENSE；社群 MCP server 授權品質參差。
- **新型污染鏈**：AI 補全/貼上 → 片段無授權標頭 → 進入金融核心 → 隱性觸發 §13。
  - 難防原因：無 metadata，傳統掃描失效，須靠**片段級（snippet）相似度比對**；潛伏期長。
- **雙面刃——用 AI 防 AI**：授權偵測助手（補 SCA 死角）、AI 加速產 SBOM、PR 階段 AI 授權守門。
- 原則：AI 是放大器；沒治理放大風險，有治理放大防禦力。

## ⑤ 風險定位 · 熱力圖與決策樹（01:05–01:20，互動）

- **熱力圖**：金融場景（內部後台→行員系統→網銀/App→開放 API→核心交易/風控）× 整合方式（純客戶端→獨立行程→動態連結→靜態/同行程）。越緊耦合、越對外，越往紅色端。
- **30 秒決策樹**：① 是否 AGPL → ② 是否對外網路服務 → ③ 是否緊耦合 → ④ 能否商業授權/替換。
- **互動橋段**：聽眾掃 QR 進 `index.html`，現場點自己系統所在格、分組各跑一條決策樹路徑並上台報告。

## ⑥ 防範縱深 + 行動清單 + Q&A（01:20–01:30）

- **六道防線**：① 授權政策（白/黑名單）② 片段級 SCA ③ CI/CD 授權門禁（fail build）④ SBOM 可追溯 ⑤ AI 工具治理（公開碼過濾＋來源宣告）⑥ 例外審批＋法務簽核。

## 講者備忘（口語要點）

- 開場節奏要快，4 個案例各 ~3 分鐘，重點在「歸納陷阱」而非細節。
- §13 段是全場核心，務必用 GPL/AGPL 對照框講清「唯一差異」。
- AI 段避免神化或妖魔化單一工具——強調「用服務 vs 散布產出」與「放大器」框架。
- 互動段預留緩衝；若時間緊，決策樹可改為全場舉手快問快答。
- 全程保持律師立場的免責：決策樹結論不可取代個案法務認定。
